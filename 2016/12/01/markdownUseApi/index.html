blog template for markdown

Posted on 2016-12-01   |   In tech

Headline: Blog template for markdown

摘要: 使用 Markdown 制作一个简单的 博客文章模板 (.md)

声明: 所有[内容]代表的意义为:内容; 实际使用请不要写括号

标题

对于一篇文章，最重要的首先是标题，markdown 定义了几种方式来自动生成常规大小的文字。

  • 第一种Setext方式:在你写好的标题下一行写上==== 或者 —-

    • 你自己文章的标题
      ==== [符号:= 的多少没有规范，但是至少大于3个，我喜欢数字4，所以我用4个]

      采用这种方式，只有2种字号大小的标题

    • 注: = 代表大标题， - 代表小标题

  • 第二种Atx方式: 这种方式定义了6种字号大小的标题，只需要在不同数目的 # 后空格再加上标题: #[空格][标题]

    • # 一级标题
      * \## 二级标题
    + \### 三级标题
        - \#### 四级标题
            * \##### 五级标题
                + \###### 六级标题
注: 在每一个块级菜单里，最大的标题之上不能有内容，否则 ==== 有可能失效, 并且 # 后空格不可少。

引用和列表

在上面博客文章标题的叙述之中，我们发现了引用和列表，下面就来探讨一下:

引用：上述标题的每一种方式，左边都有一个竖线来按照块级包裹，这就是引用。
引用的使用方式为：
>[空格][引用内容]
多层级引用使用多个 > (格式如下：)
>>[空格][引用内容]

列表 (多层列表嵌套使用缩进)

无序列表

-[空格][内容]
-[空格][内容]

有序列表

1.[空格][内容]
2.[空格][内容]

注: 数字和点之间不要空格，再引用之后使用有时不生效
注: - + * 这三个符号在用作无序列表时作用功能一致，多层嵌套建议切换使用

字体大小与样式

列表之后就是写正文了，关于正文，无非就是字体大小与样式。

斜体
将需要设置为斜体的文字两端使用1个“”或者“_”夹起来
格式Example: \[内容]* 或者 -[内容]-

粗体
将需要设置为斜体的文字两端使用2个“”或者“_”夹起来
格式Example: \*[内容]** 或者 --[内容]--

样式：色彩 | 字号 | 字体

方式一:

<font color="色彩代码"></font>
<font size="4"></font>
<font face="微软雅黑"></font>

方式二:

<font style="color: '色彩代码'; font-size: 20px; font-family: '微软雅黑'"></font>

注: 第二种的font-size后的数值要带单位 px

超链接 (图片, 网站)

接下来在我们文章书写之中，如果遇到了需要引用外部链接或者图片的情况呢？

More info: [home](https://isdux.com)

如此写法，就会将文字 home 指向一个外部的链接，效果如下:

More info: home

如果是图片的话，我们只需要将上面的代码稍作修改

[home](https://isdux.com) 格式改为:
![home]("https://isdux.com "home")
将 https://isdux.com 更换为真实的图片地址之后
![home](https://assets-cdn.github.com/images/modules/site/home-ill-build.png?sn"home")

前面加一个感叹号，后面加一个双引号

这样就实现了对 github 官网上一个图片的引用。

可是如果这个图片是某个项目文件里自带的呢？

![Some_Text](url_to_image "Some_Text")

那么同样的，只需要 url_to_image 换成相对的路径(/path/to/img.jpg)
注: 这里的 path 代表当前文件的位置。

代码

特殊的，我们作为技术达人在分享代码时候，如何和其他文字区分？
我们以一个基本的 html dom 为例，这有两种后方式:

第一种:
采用两个`` (注: 这里的`不是分号，而是 Exc 下面的~的英文输入形式)
格式: ‘[代码内容]’

类似如下格式书写:
`<!DOCTYPE html>`

页面输出为:
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>Example</title>
</head>
<body>
<a href=""></a>
</body>
</html>

第一种方式通常应用在单行或者少量代码的时候使用。

我们注意到，这种方式没有高亮和缩进，可是如果我想展示很好的缩进并且自动高亮呢？
那么我们就必须采用第二种方式：
这就需要 换行， tab 或者 space。

首先，必须出现一个空白行，然后在第二行使用 tab, 然后就可以输入你要输入的代码，在markdown 编辑器里文字会自动变成灰色块区。
然后，你也可以使用 space 控制，不过，这里你需要 4个 space 才能达到 tab 的效果。默认回车时缩进为上一行设置的。
注: 代码上一行必须为空白或者 — 等无意义的标识。

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Example</title>
</head>
<body>
    <a href=""></a>
</body>
</html>
第二种方式通常应用在多行或者大量代码引用时。

删除线

最后，我们如果遇到某个需要告诉大家已经落伍的或者淘汰了的技术，我们可以使用删除线。

<del>Underlined Text</del>

同样的，我们需要类似宣传最新技术的时候需要下划线引起注意，我们可以使用:
