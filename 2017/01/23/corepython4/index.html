因特网客户端简介

当今因特网中大部分客户端/服务器协议的核心，是使用套接字的底层网络通信协议。

这些网络通信协议分别用于：

  • 文件传输 (FTP 、SCP 等)
  • 阅读 (Usenet)
  • 新闻组 (NNTP)
  • 发送电子邮件 (SMTP)
  • 从服务器上下载电子邮件 (POP3 、IMAP)
  • … …

因特网： 用来传输数据的地方，数据在服务提供者和服务使用者之间传输。在某种情况下，我们也成为 “ 生产者 – 消费者 ”。
服务器就是生产者，提供服务，而客户端使用服务。
对于特定的服务，一般只有一个服务器/主机/进程，但有多个消费者。

文件传输

文件传输因特网协议

主流的文件传输的协议:

  • 文件传输协议 (FTP)
  • UNIX 到 UNIX 复制协议 (UUCP)
  • 用于 web 的超文本传输协议 (HTTP)
  • (UNIX下的)远程文件复制命令 rcp
  • 更安全更灵活的 scp 和 rsync

主流的文件传输协议的主要用途：

  • HTTP 主要用于基于 Web 的文件下载以及访问 Web 服务，一般客户端无需登陆就可以访问服务器上的文件和服务。大部分 HTTP 文件传输请求都用于获取网页(即将网页文件下载到本地)
  • scp 和 rsync 需要用户登陆到服务器主机。在传输文件之前必须验证客户端的身份，否则不能上传和下载文件。
  • FTP 与 scp/rsync 类似，也可以上传和下载文件，并采用了 UNIX 的多用户概念，用户需要输入有效的用户名和密码。但 FTP 协议允许匿名登陆。

文件传输协议 FTP

File transfer protocol

FTP 主要用于:

  • 匿名下载公共文件
  • 两台计算机之间的传输文件
  • windows 进行工作，文件存储系统使用 UNIX
anonymous [ə'nɒnɪməs] 匿名的，无名的，无个性特征的，匿名者

在底层，FTP 只使用 TCP，而不适用 UDP。它使用连个套接字来通信。

  • 控制和命令端口(21端口)
  • 数据端口

FTP 有两种模式，主动和被动。

  • 主动模式下才使用数据端口20
  • 被动模式下使用随机数据端口N > 1023
