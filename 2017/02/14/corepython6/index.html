(6) python 核心编程

Posted on 2017-02-14   |   In tech

Headline: [6]Core python programming

摘要: 数据库编程

声明: 本篇博文为本人 python 核心编程的读书笔记

数据库编程

持久化存储以及数据库系统

持久化存储

关于持久化存储，一般有3种基础的存储机制。

  • 文件(json, txt, csv, xml…)
  • 数据库系统(关系型, 非关系型)
  • 混合类型(对象关系映射, API, 文件管理器, 电子表格, 配置文件…)
数据库基本操作和sql

底层存储：
数据库通常使用文件系统作为基本的持久化存储，他们可以是普通的操作系统文件、专用的操作系统文件、甚至是原始的磁盘分区。

用户接口：
通常是命令行借口。

数据库：
关系型数据库管理系统(RDBMS – Relational Database Management System)

  • MySQL 基于服务器
  • SQLite/Gadfly 不运行服务器

组件:
数据库存储可以抽象为一个表。每行数据都有一些字段对应于数据库的列。每一列的表定义的集合以及每个表的数据类型放到一起定义了数据库的模式。(schema)

python 和 数据库

python 访问关系型数据库：

  • 直接通过数据库接口访问
  • 使用 ORM(Object Relational Mapping) 访问

其中，ORM访问的方式不需要显示给出SQL命令，也能完成相同的任务。

数据库常见知识类别:

  • 数据库原理：
  • 并发性
  • 模式
  • 原子性
  • 完整性
  • 恢复
  • 比较法咋的左连接
  • 触发器
  • 查询优化
  • 事务性
  • 存储过程

完全用 python 编写的简单的 RDBMS (Gadfly)

在python中数据库是通过适配器的方式进行访问的。适配器是一个python模块，使用它可以与关系数据库的客户端库接口相连。

python 的 DB-API

DB-API 是阐明一系列所需对象和数据库访问机制的标准，他可以为不同的数据库适配器和底层数据库系统提供一致性的访问。DB-API 也是强需求驱动的。

DB-API 模块属性:

  • apilevel: 需要适配器兼容的 DB-API 版本
  • threadsafety 本模块的线程安全级别
  • paramstyle 本模块的SQL语句参数风格
  • connect() Connect()函数
  • Exception (多种异常)
Connection 对象

当以连接(或一个连接池)建立后，可以创建一个游标，向数据库发送请求，然后从数据库中接收回应。

rollback() 只有在支持事务处理的数据库中才有用。 – 关闭连接而不事先提交变更，将会导致执行隐式回滚

Cursor 对象

创建一个数据库适配器，还必须要实现cursor对象，以扮演类似游标的角色。这样，无论数据库系统是支持或不支持游标，都能保证python代码的一致性。

类型对象与构造函数

为了与数据库对象一一对应

关系数据库

ORM

目前，知名的Python ORM 是SQLAlchemy 和 SQLObject, 其他的还包括： Strom, PyDO/PyDO2, PDO, Dejavu, Durus, QLime, ForgetSQL等。

非关系数据库 / NoSQL
  • MongoDB
  • Redis
  • Cassandra
  • SimpleDB
  • Tokyo Cabinet
  • CouchDB
  • Bigtable (通过 Google App Engine 的数据库 API)
  • 对象数据库
  • 键-值对存储
  • 文档存储(或数据存储)
  • 图形数据库
  • 表格数据库
  • 列/可扩展记录/宽列数据库
  • 多值数据库
MongoDB

MongoDB属于NoSQL数据库，使用文档数据存储(c++编写的文档存储系统)，与其他非关系数据库相比，它介于简单的键-值对存储(如 Redis, Voldemort, Amazon Dynamo等) 与列存储(如 Cassandra, Google Bigtable 和 HBase) 之间。

特点:

  • 比基于列的存储更简单、约束更少
  • 比普通的键-值对存储更加灵活
  • 一般情况下其数据会另存为JSON对象
  • 允许诸如字符串，数值，列表甚至嵌套等数据类型
